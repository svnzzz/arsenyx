# CLAUDE.md — apps/web

Vite + React 19 + TanStack Router (file-based) + TanStack Query + Tailwind v4 + shadcn/ui (Base UI underneath).

## Conventions

- Route with detail page that should NOT inherit parent layout: trailing underscore (`browse_.$category.$slug.tsx`). Breaks layout nesting, keeps URL clean.
- All internal links go through [src/components/link.tsx](src/components/link.tsx) — wraps TanStack `Link`, falls back to `<a>` for external URLs.
- Shared warframe types / client helpers live in [src/lib/warframe.ts](src/lib/warframe.ts). Heavy data is static JSON under `public/data/`, generated by [scripts/build-items-index.ts](../../scripts/build-items-index.ts) (`bun run build:items`).
- **Two build entries** ([vite.config.ts](vite.config.ts) `rollupOptions.input`): the full SPA (`index.html` → `src/main.tsx`) and a router-less read-only viewer (`embed.html` → [src/embed-main.tsx](src/embed-main.tsx)). For `/builds/:slug?embed=1` the Worker ([worker/index.ts](worker/index.ts)) serves `embed.html` instead of the SPA, so a guide page's many iframes don't each boot the whole app. The embed must stay off the router/app-shell graph: import viewer pieces via deep paths (not the `@/components/*` barrels, which fail to tree-shake) and the embed fetches the build with `?view=0` (no view-count, browser-cacheable — see [build-query.ts](src/lib/queries/build-query.ts) + api `builds.ts`).

For TanStack Router specifics (loaders, search params, layout nesting) consult the upstream [docs](https://tanstack.com/router/latest) when needed.
